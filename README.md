## 📂 Featured Projects

<details>
<summary><b>📖 DocTalk – RAG-Based PDF Chatbot</b></summary>
<br />

#### Business Problem
Manual analysis and semantic searching within large enterprise PDF documents is highly inefficient, leading to wasted operational hours and critical information loss.

#### Solution
Built a full-stack Retrieval-Augmented Generation (RAG) micro-app integrating FastAPI, ChromaDB, and Groq-hosted LLMs, featuring chunking optimization, vector indexing, and fallback layers.

#### Impact
Enabled low-latency natural language interaction over uploaded PDF records with real-time response streaming, reducing query extraction time by over 90%.

</details>

<details>
<summary><b>🎥 YouTube ChatBot – RAG-Based Video Q&A Assistant</b></summary>
<br />

#### Business Problem
Retrieving specific insights, citations, and timestamps from hour-long YouTube tutorials and technical recordings is highly time-intensive.

#### Solution
Developed a full-stack ingestion utility that downloads YouTube transcriptions, segments and embeds them using BAAI/bge-m3 embeddings, and indexes them inside a local FAISS database for conversational Q&A via OpenRouter.

#### Impact
Delivered contextual, conversational answers alongside precise chunk-level citations and response times, saving up to 80% of video scanning time.

| Stack | Architecture | Performance | Security | Impact | Repository | Lessons Learned |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| Python, Streamlit, LangChain, FAISS, OpenRouter (Qwen 2.5 7B) | Distributed RAG / Embedded Vector Database | Response-time tracking and low-latency embeddings | Secure transcript sanitization | 80% time saved on video reviews | [Repository Link](https://github.com/KhushTrivedi445/YouTube-ChatBot) | Tracking response-time helps in validating API bottlenecks |
